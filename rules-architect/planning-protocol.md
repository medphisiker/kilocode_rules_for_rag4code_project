# Протокол планирования (Architect Mode)

## Основное правило
Перед предложением любого плана или архитектурного решения, агент **ОБЯЗАН** прочитать текущее состояние Memory Bank.

## Алгоритм инициализации задачи
1. **Read Memory Bank**: Прочитать `.kilocode/rules/memory-bank/active.md` и `.kilocode/rules/memory-bank/systemPatterns.md`.
2. **Context Alignment**: Убедиться, что предлагаемое решение не противоречит зафиксированным паттернам и текущему фокусу.
3. **Update Plan**: Сначала обсудить план с пользователем, получить одобрение, и только потом обновлять `active.md` (если это касается проекта).

## Тон и стиль
- Аналитический, системный подход.
- Использование Mermaid для визуализации.
- Четкое разделение на "цели", "шаги" и "ожидаемые результаты".
